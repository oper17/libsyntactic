The C++ Neural Network Library Users Documentation


This documentation is for users prpose only. Developers too, are recommended to read this documentation first before the Developers Documentation.

    The following steps are to be follwed:
      
    Step 1: Create a network with 'n' number of layers
         
    Use the:
          constructor VVNetwork( int arg_layers, int neuro_layer_1, int neuro_layer_2, ... , int neuro_layer_n, double learning_rate, double momentum, int iter);
             Where :
      • int arg_layers = Number of layers in the network.
      • int neuro_layer_1 = Number of neurons in first layer.
      • int neuro_layer_2 = Number of neurons in second layer.
      • ...
      • upto ... int neuro_layer_n = Number of neurons in the nth layer(last layer).
      • double learning_rate = The learning rate of the network
      • double momentum = The momentum of the network.
      • int iter = The number of iterations to train for.  (Note: setting iter to a value, will not cause training for that many iterations. The iterations must be done externally using a loop.)

             For example for a 3 layer, 2 input network to simulate an xor gate:
   
                   VVNetwork xorgate ( 3, 2, 2, 1, 0.87, 0.2, 1000);

    Step 2: Set the transfer functions


    To do this we do:
         
             xorgate.vvtransfer_function = VV_SIGMOID_HALF;
            xorgate.vvtransfer_function_derivative = VV_SIGMOID_HALF_DERIV;

          Here, we have selected the VV_SIGMOID_HALF & VV_SIGMOID_HALF_DERIV for the transfer functions.
          The other available transfer functions are:

      • VV_HARDLIMIT & VV_HARDLIMIT_DERIV
      • VV_SIGMOID_FULL & VV_SIGMOID_FULL_DERIV
      • VV_RAMP & VV_RAMP_DERIV
      • VV_HYPERBOLIC & VV_HYPERBOLIC_DERIV

    Step 3: Select the input data file and the actual desired output file
   
    We use:
         
             xorgate.VVreadinputfrom("xor.in");
             xorgate.VVreadoutputfrom("xor.out");

             Where xor.in contains: (In the case of 2 input xor)
                0   0
                0   1
                1   0
                1   1

               And xor.out contains:(In the case of xor):
                0
                1
                1
                0

    Step 4: Set biasing if required

          To do that, use:
      
                   xorgate.vvautobias()

          This automatically biases all non-input neurons with inital random weights.

    Step 5: Training the network

          To train, we simply use:

               for(int i=0; i< 1000; i++)
                xorgate.VVnetwork_train();

          This trains the network for 1000 iterations, note the number of iterations in the loop must match the number passed to the constructor as iterations.

    Step 6: Saving the network

          The network is saved into a file, so that it can be used later as per requirements by simply loading it.

                   xorgate.VVsave_network("xor.nnw");


    Step 7: Loading the network

       The network can be loaded any time later for use.
   
                VVNetwork xorgate2;
                  xorgate2.VVload_network("xor.nnw");

       Next we have to set the transfer functions. This is done as in step 2.


    Step 8: Testing the network

    To test the network, we must specify the test file and a vector of doubles to hold the network's output.

             vector<double> out;
             out=N2.VVtest_network("test");
      
            For eg.
                    contents of test:
                     0 1
            
    Now the out vector holds the output, in this case(xor), it holds just one output and it is usually around 0.91 - 0.99 approx = 1 (for this example).

    Step 9: Processing the output

    Now the dealing with the entire network is complete. The output is stored in vector<double> out.
    In the above case of an xor, just printing the contents of out[0] will be enough, but for more complicated networks used for purposes such as
    pattern recognition etc, some post processing must be done to ensure comprehensible output for normal users.
